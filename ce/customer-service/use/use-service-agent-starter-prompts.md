# Use starter prompts in Service Agent

Starter prompts are suggested prompts that appear when you open Service Agent or start a new conversation. They help you quickly perform common tasks and automatically include details from the current case when applicable.

## Prerequisites

Your administrator must configure Service Agent before starter prompts are available.

Learn more in [Enable Service Agent](../administer/configure-service-agent.md).

## Use a starter prompt

1. Open Service Agent.
1. Select a starter prompt.
1. Review the prompt and make any changes if needed.
1. Press **Enter** or select **Send**.

Starter prompts can automatically include information from the current case, such as the case number and title.

For example:

**Case Summary**
*What is the summary of my case CAS-01135-Y4K2P8: Replace non-working device?*

## View more starter prompts

If more starter prompts are available, select **See more** to expand the full list.

## Use organization starter prompts

Your administrator can configure custom starter prompts for your organization. These prompts appear alongside the default starter prompts and are tailored to your team's workflows.

## Considerations

- Starter prompts work best when a case is open so Service Agent can automatically include case details.
- You can edit a starter prompt before sending it.
- The starter prompts that are available depend on how your administrator configured Service Agent.
